  用户输入
      │
      ▼
  【1. 接受输入】
  packages/stage-ui/src/stores/chat.ts
    └─ ingest(message, options)
         └─ sendQueue.enqueue(...)  ← 排队，防止并发冲突
              └─ performSend(message, options, generation, sessionId)

  【2. 构建消息上下文】
    └─ chatSession.getSessionMessages(sessionId)
         ↑ packages/stage-ui/src/stores/chat/session-store.ts
         │  返回 [systemMessage, ...历史消息]
         │  systemMessage 来自角色卡：useAiriCardStore().systemPrompt
         │
    └─ chat.ts 强制替换第一条 system 消息为 liveSystemPrompt.value（本次修复）
    └─ buildContextPromptMessage(contexts)  ← 注入日期时间等上下文
    └─ newMessages = [system, contextMsg?, ...history, userMsg]

  【3. 发送 AI 请求】
    └─ llmStore.stream(model, chatProvider, newMessages, options)
         ↑ packages/stage-ui/src/stores/llm.ts
         └─ streamFrom() → xsai streamText()  ← 调用 OpenAI 兼容 API
              SSE 流式返回 text-delta 事件

  【4. 解析响应流】
    └─ onStreamEvent → onLiteral / onSpecial（特殊 token）
         ↑ useLlmmarkerParser
         │  packages/stage-ui/src/composables/llm-marker-parser.ts
         │  识别 <|ACT:...|> 为 special，其余为 literal
         │
    └─ createStreamingCategorizer
         packages/stage-ui/src/composables/response-categoriser.ts
         识别 <think>...</think> 等推理 tag，过滤掉不发给 TTS
         若是 JSON 格式回复，提取 reply 字段 + 触发 emotion 动画

  【5. 回复给玩家】
    ├─ 文本 → buildingMessage.content 累加 → UI 实时更新
    ├─ TTS → hooks.emitTokenLiteralHooks(speechOnly)
    │         → Stage.vue onTokenLiteral → speechPipeline.writeLiteral()
    │         → tts-chunker.ts 按标点分句
    │         → TTS API 合成语音 → 播放
    └─ 情绪/动画 → hooks.emitTokenSpecialHooks(special)
                  → Stage.vue onTokenSpecial → parseActEmotion()
                  → currentMotion.value → Live2D 动作表情

---
  这次问题根因：performSend 用的 system
  消息是会话创建时存下来的快照，可能在角色卡还没加载时就已经固化为空字符串。现在的修复在 performSend
  发送前直接用 liveSystemPrompt.value（从 useAiriCardStore 实时读取）覆盖第一条 system
  消息，任何时序问题都绕过了。
