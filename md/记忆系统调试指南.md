# AIRI 记忆系统调试指南

## 概述

AIRI 的记忆系统基于 Claude Code 的 memdir 架构，实现了跨平台的长短期记忆功能。本指南帮助开发者调试和理解记忆系统的运行情况。

## 启用调试日志

### 方法1：开发模式自动启用
在开发环境（`import.meta.env.DEV = true`）下，记忆系统日志会自动启用。

### 方法2：手动启用
在生产环境或需要强制启用时，在浏览器控制台执行：

```javascript
localStorage.setItem('debug:memory', 'true')
// 然后刷新页面
location.reload()
```

### 禁用调试日志
```javascript
localStorage.removeItem('debug:memory')
// 或设置为 false
localStorage.setItem('debug:memory', 'false')
```

## 日志模块

记忆系统的日志分为以下几个模块，每个模块有独立的前缀：

### 1. `[MemoryStore]`
核心 Pinia store 的日志，包含：
- `init:` - 初始化过程
- `upsertMemory:` - 创建/更新记忆
- `deleteMemory:` - 删除记忆
- `buildMemoryPrompt:` - 生成系统提示
- `extractFromConversation:` - LLM 提取记忆

**示例输出：**
```
[MemoryStore] init: loading memories for character: default
[MemoryStore] init: loaded 3 memories [{...}, {...}, {...}]
[MemoryStore] init: existing index length: 156
[MemoryStore] init: complete, ready=true

[MemoryStore] upsertMemory: user 用户偏好
[MemoryStore] upsertMemory: created new memory: abc123
[MemoryStore] upsertMemory: total memories now: 4
```

### 2. `[MemoriesRepo]`
数据访问层日志，包含：
- `listByCharacter:` - 查询角色的所有记忆
- `getById:` - 根据 ID 查询单条记忆
- `upsert:` - 保存记忆到数据库
- `delete:` - 从数据库删除记忆
- `rebuildIndex:` - 重建记忆索引

**示例输出：**
```
[MemoriesRepo] listByCharacter: default
[MemoriesRepo] listByCharacter: returned 4 records
[MemoriesRepo] upsert: user 用户偏好
[MemoriesRepo] upsert: success, id: abc123
```

### 3. `[MemoryContext]`
上下文注入日志，包含：
- `createMemoryContext:` - 创建记忆上下文消息

**示例输出：**
```
[MemoryContext] createMemoryContext: injecting memory context, length: 1234
[MemoryContext] buildMemoryPrompt: memory counts by type: {user: 2, feedback: 1, project: 0, reference: 1}
[MemoryContext] buildMemoryPrompt: generated 1234 bytes, 45 lines
```

### 4. `[ChatOrchestrator]`
聊天流程日志，包含：
- Session 切换时的记忆提取触发

**示例输出：**
```
[ChatOrchestrator] Session switch detected, triggering memory extraction: {oldId: "xxx", newId: "yyy", messageCount: 15}
[ChatOrchestrator] Starting memory extraction with model: gpt-4
[ChatOrchestrator] Memory extraction completed
```

## 调试场景

### 场景1：检查记忆是否被保存
1. 启用调试日志
2. 触发 session 切换（切换到另一个会话再切回来）
3. 查找以下日志序列：
   ```
   [ChatOrchestrator] Session switch detected...
   [ChatOrchestrator] Starting memory extraction...
   [MemoryStore] extractFromConversation: starting with X messages
   [MemoryStore] extractFromConversation: calling LLM for extraction...
   [MemoryStore] extractFromConversation: saved N/M memories
   ```

### 场景2：检查记忆是否被注入到聊天
1. 启用调试日志
2. 发送一条新消息
3. 查找以下日志：
   ```
   [MemoryContext] createMemoryContext: injecting memory context, length: XXX
   ```
   如果返回 `no memories to inject`，说明当前角色没有保存的记忆

### 场景3：验证数据库连接
1. 启用调试日志
2. 刷新页面
3. 查找初始化日志：
   ```
   [MemoryStore] init: loading memories for character: XXX
   [MemoriesRepo] listByCharacter: XXX
   ```
   如果有错误，会显示 `[MemoryStore] init: failed to load memories` 或 `[MemoriesRepo] listByCharacter: failed`

### 场景4：检查记忆索引重建
1. 当记忆数量变化时，索引会自动重建
2. 查找日志：
   ```
   [MemoriesRepo] rebuildIndex: default
   [MemoriesRepo] rebuildIndex: success, total records: X
   ```

## 常见问题排查

### Q: 没有看到 `[MemoryStore]` 日志
**A:** 检查调试模式是否启用：
```javascript
console.log(localStorage.getItem('debug:memory'))
// 应该输出 "true"
```

### Q: 看到 `failed to load memories` 错误
**A:** 可能是数据库适配器未正确注册。检查对应平台的 main.ts：
- **stage-pocket**: `setupDatabase()` 是否被调用
- **stage-tamagotchi**: `registerDatabaseAdapter()` 是否被调用
- **stage-web**: 应该自动使用 IndexedDB

### Q: Session 切换后没有触发记忆提取
**A:** 检查：
1. LLM provider 和 model 是否配置
2. 查找 `[ChatOrchestrator] No chat provider available` 或 `No model configured` 警告
3. 确保旧 session 有足够消息（至少 3 条）

### Q: 记忆提取返回空数组
**A:** 这是正常的。LLM 可能认为当前对话没有值得长期记住的信息。查看日志：
```
[MemoryStore] extractFromConversation: no memories extracted (empty array)
```

### Q: 记忆内容被截断
**A:** 这是设计行为，遵循 Claude Code 的限制：
- 最大 200 行
- 最大 25 KB
超出时会显示：
```
[MemoryStore] buildMemoryPrompt: content too large (XXX bytes), falling back to index-only
[MemoryStore] buildMemoryPrompt: too many lines (XXX), truncating to 200
```

## 手动测试记忆功能

在浏览器控制台执行以下代码进行手动测试：

```javascript
// 获取 memory store
import { useMemoryStore } from '@proj-airi/stage-ui/stores/memory'

const memoryStore = useMemoryStore()

// 查看当前记忆数量
console.log('Total memories:', memoryStore.records.length)

// 按类型查看
console.log('By type:', memoryStore.byType)

// 手动添加测试记忆
await memoryStore.upsertMemory({
  name: '测试记忆',
  description: '这是一个测试记忆',
  type: 'user',
  content: '用于测试记忆系统是否正常工作'
})

// 生成记忆提示
const prompt = memoryStore.buildMemoryPrompt()
console.log('Memory prompt:', prompt)

// 刷新后验证持久化
location.reload()
// 然后再次检查记忆数量
```

## 性能监控

记忆系统的关键性能指标：
- **初始化时间**: `init:` 调用时间
- **提取时间**: `extractFromConversation` 完成时间
- **索引大小**: 索引字节数和行数

如需监控性能，可在 Chrome DevTools Performance 面板记录包含记忆操作的交互。

## 平台特定日志

### Electron (stage-tamagotchi)

```

```

### 移动端 (stage-pocket)
