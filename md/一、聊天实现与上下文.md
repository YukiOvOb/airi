  一、聊天实现与上下文

  核心数据流

  用户输入
    → useChatOrchestratorStore.send()        packages/stage-ui/src/stores/chat.ts
      → 构建消息 + 注入上下文快照              stores/chat/context-store.ts
      → 触发钩子 onBeforeMessageComposed
      → useLLM.stream()                       stores/llm.ts
        → 流式事件: text-delta / tool-call / finish
        → LLM marker解析器 提取情感/延迟标记   composables/llm-marker-parser.ts
      → 触发钩子 onStreamEnd, onAssistantResponseEnd
      → 追加消息 + 持久化到 IndexedDB         stores/chat/session-store.ts

  上下文组成

  - System prompt = AIRI角色卡 + 上下文快照（时间、外部模块数据）
  - 历史消息 = session中所有 user/assistant/tool messages
  - 工具列表 = spark-command工具 + MCP工具（动态注入）

  支持的LLM Provider

  OpenAI、Azure OpenAI、Anthropic、Google、OpenRouter、Ollama、DeepSeek、OpenAI-compatible

  如何替换Live2D人物

  方式1：通过设置UI
  - 设置页 → 模型管理 → 上传ZIP文件或输入URL
  - display-models.ts 存入IndexedDB
  - stage-model.ts 中 stageModelSelectedUrl 更新
  - Live2DScene prop model-src 变化 → Model.vue 重新加载

  方式2：直接替换文件
  - 将 .model3.json 和相关资源打包成ZIP
  - 通过 displayModelsStore.addDisplayModel() 导入

  模型要求
  - 标准Cubism 4 格式（.model3.json）
  - 动作组名称决定情感映射（EMOTION_EmotionMotionName_value 中配置）
  - ReLU模型当前映射：Tap / Flick@Body / Tap@Body / FlickDown / FlickUp / Flick / Think / Surprise / Idle / Awkward

  三、聊天与Live2D互动

  情感驱动流程

  LLM输出: "你好！<|ACT:{emotion: 'happy', intensity: 0.8}|>"
    ↓
  llm-marker-parser 检测到 ACT 标记
    ↓
  useEmotionsMessageQueue.parse() → { name: 'happy', intensity: 0.8 }
    ↓
  Stage.vue 中的 watcher 触发
    ↓
  EMOTION_EmotionMotionName_value['happy'] = 'Tap'
    ↓
  currentMotion.value = { group: 'Tap' }
    ↓
  Model.vue 的 watch(currentMotion) → live2DModel.motion('Tap', index)

  语音同步流程（嘴型）

  TTS生成音频
    → createLive2DLipSync()              model-driver-lipsync包
      → 每帧读取音量/振幅
      → 更新 Live2D ParamMouthOpenY 参数

  Stage.vue 是核心编排点

  packages/stage-ui/src/components/scenes/Stage.vue
  - 订阅聊天钩子（onTokenLiteral, onAssistantResponseEnd）
  - 处理情感队列 → 触发Live2D动作
  - 处理延迟队列 → 暂停TTS播放
  - BroadcastChannel跨标签同步（airi-emotion-test频道）

  四、其他功能架构

  语音系统

  STT: 麦克风 → VAD检测 → hearingPipeline.transcribeForRecording() → 文字 → 自动发送
           stores/ai/models/vad.ts (TensorFlow WebAudio)
           stores/modules/hearing.ts (Aliyun/Deepgram/WebSpeech)

  TTS: 文字 → useSpeechRuntimeStore.openIntent() → TTS Provider → 音频播放 → 嘴型同步
           stores/speech-runtime.ts
           stores/modules/speech.ts (ElevenLabs/Microsoft/CozyVoice/Kokoro)

  Mod/插件系统

  WebSocket → useModsServerChannelStore      stores/mods/api/channel-server.ts
    → useContextBridgeStore                  stores/mods/api/context-bridge.ts (API暴露给插件)
    → 事件路由 channel-gateway.ts
    → 支持事件: spark:notify, spark:command, ui:configure

  角色编排系统（Spark）

  spark:notify 事件 (任务提醒/Discord消息等)
    → useCharacterOrchestratorStore
      → Spark Notify Agent (LLM处理通知)
      → 生成反应文本 → TTS
      → 发出 spark:command 给子Agent
