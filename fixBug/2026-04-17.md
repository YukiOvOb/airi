# Bug 修复记录 — 2026-04-17

## 背景

在将 AIRI 项目适配手机端（移动浏览器 + Capacitor）的过程中，发现并修复了多个问题。

---

## 修复内容

### 1. 对话 API 400 错误（schema 不合法）

**文件：** `packages/stage-ui/src/tools/character/orchestrator/spark-command-shared.ts`

**问题：** `normalizeNullableAnyOf` 函数在处理 `{ "const": false }` 这类条目时，没有补充 `type` 字段，导致发送给 OpenAI 的 JSON Schema 不合法，API 返回 400。

**修复：** 检测 `const` 值类型，自动补 `type: "boolean" | "string" | "number"`。

---

### 2. 手机端无法加载角色（DuckDB 崩溃阻塞初始化）

**文件：** `apps/stage-pocket/src/App.vue`

**问题：** 手机端不支持 DuckDB WASM，初始化时崩溃，导致后续的 `cardStore.initialize()` 无法执行，角色数据为空，Live2D 无法加载。

**修复：** 在 `onMounted` 中所有 `await` 调用加 `.catch()` 隔离，防止单点失败阻断整体流程。

---

### 3. wlipsync 在移动端崩溃（AudioWorkletNode 模块级报错）

**文件：** `packages/model-driver-lipsync/src/live2d/index.ts`

**问题：** `wlipsync` 在模块顶层引用了 `AudioWorkletNode`，在手机 Safari/WebView 中初始化时直接抛出异常，导致整个 Live2D 模块加载失败。

**修复：** 将 `import { createWLipSyncNode } from 'wlipsync'` 改为函数内动态 `import()`，延迟到实际调用时才加载。

---

### 4. Live2D 从设置页返回后消失

**文件：**
- `packages/stage-layouts/src/layouts/stage.vue`
- `apps/stage-pocket/src/App.vue`
- `packages/stage-ui/src/components/scenes/Stage.vue`
- `packages/stage-ui-live2d/src/components/scenes/live2d/Canvas.vue`

**问题：** 从设置页返回主界面后，Live2D 模型不再显示。根因是：
- `KeepAlive` 通过组件 `name` 匹配，但 `stage.vue` layout 没有 `name` 声明，导致缓存失败；
- Canvas.vue 没有处理 WebGL context 丢失（移动端 Safari 在导航切换时会回收 WebGL 上下文）；
- Stage.vue 没有在重新激活时重新加载模型。

**修复：**
- `stage.vue` 添加 `export default { name: 'StageLayout' }`；
- `App.vue` 的 `KeepAlive` 改为 `:include="['StageLayout']"`；
- `Canvas.vue` 添加 `onActivated`，检测 WebGL context 是否丢失，丢失则重建；
- `Stage.vue` 添加 `onActivated`，强制重新调用 `updateStageModel()`，相当于刷新重载。

---

### 5. Beat-sync 在 Capacitor 环境报错

**文件：** `packages/stage-shared/src/beat-sync/detector.ts`

**问题：** `getDetector()` 内部有 `if (!isStageWeb()) throw new Error(...)` 的环境检查，Capacitor 运行时（`RUNTIME_ENVIRONMENT = 'capacitor'`）不属于 Web，导致所有 beat-sync 监听函数报错，进而阻断 Live2D 模型初始化流程。

**修复：**
- 将检查改为 `if (!isStageWeb() && !isStageCapacitor()) throw new Error(...)`；
- 同时修正 detector 创建时的 env 参数：Capacitor 环境使用 `StageEnvironment.Capacitor`。
- 所有外部导出函数（`toggleBeatSync`、`listenBeatSyncBeatSignal` 等）也同步补充 `isStageCapacitor()` 条件。

---

### 6. 设置页按钮不可见（v-motion 初始 opacity 为 0）

**文件：** `packages/stage-ui/src/components/layouts/ripple-grid/index.vue`

**问题：** `RippleGrid` 组件用 `v-motion` 给每个 item 做入场动画（`initial: { opacity: 0, y: 10 }`），在移动端 `enter` 动画不触发，导致所有按钮一直保持 `opacity: 0`——看不见但仍可点击。

**修复：** 将默认 `animationInitial` 从 `{ opacity: 0, y: 10 }` 改为 `{ opacity: 1, y: 10 }`，确保 item 始终可见，保留 y 轴滑入动画。

## 项目整体结构

```
AIRI/
├── apps/
│   ├── stage-web/          # 浏览器 Web 版（Vue 3 + Vite）
│   ├── stage-pocket/       # 手机 App 版（Capacitor + Vue 3）
│   ├── stage-tamagotchi/   # 桌面版（Electron + Vue 3）
│   └── ...
│
├── packages/
│   ├── stage-ui/           # 核心 UI 组件、Pinia stores、composables（各端共用）
│   ├── stage-ui-live2d/    # Live2D 渲染组件（PixiJS + pixi-live2d-display）
│   ├── stage-ui-three/     # Three.js / VRM 渲染组件
│   ├── stage-shared/       # 跨端共用逻辑（beat-sync、环境检测等）
│   ├── stage-layouts/      # 页面布局组件（stage、settings、home 等 layout）
│   ├── stage-pages/        # 共用页面（settings 下各子页）
│   ├── model-driver-lipsync/ # Live2D 口型同步驱动（wlipsync）
│   ├── ui/                 # 基础 UI 原语（基于 reka-ui）
│   ├── i18n/               # 多语言翻译
│   └── ...
│
├── services/               # 后端服务（Node.js）
├── plugins/                # 插件系统
├── integrations/           # 第三方集成
└── scripts/                # 构建脚本
```

| 口型同步 | wlipsync（AudioWorklet）|
| Beat 检测 | @nekopaw/tempora（AudioWorklet）|

### 环境判断机制

`packages/stage-shared/src/environment.ts` 通过 `import.meta.env.RUNTIME_ENVIRONMENT` 区分运行时：

- `'web'`（默认）→ `isStageWeb() = true`
- `'capacitor'` → `isStageCapacitor() = true`（stage-pocket 在 vite.config.ts 中强制注入）
- `'tamagotchi'` → `isStageTamagotchi() = true`（Electron 渲染进程）

手机端测试时通过局域网访问 Vite dev server（`host: 0.0.0.0`，端口 5273）。
